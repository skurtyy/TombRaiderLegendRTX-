# Build 075 — Replacement Assets Fix + Stale Hash Discovery

## Result
**FAIL-stale-hashes** — Major breakthrough: replacement asset pipeline now works (purple light visible). Stage lights absent because the 8 building mesh hashes in mod.usda are stale — captured under a previous Remix configuration.

## What Changed This Build
1. **CRITICAL FIX: `user.conf` override** — `rtx.enableReplacementAssets = False` in `user.conf` was silently disabling ALL mod content (lights, materials, meshes). Changed to `True`. This single line was the root cause of lights being absent in builds 016-074.
2. **Fallback light dimmed** — radiance 100→5 to let colored mod lights dominate
3. **Tested both `useVertexCapture` modes** — False (matching original hash capture) and True. Neither produces visible red/green lights.
4. **Tested light radius from 2 to 3000 game units** — no change in appearance, confirming the mesh hashes don't match any currently rendered geometry.

## Root Cause Analysis
The `user.conf` in the game directory had `rtx.enableReplacementAssets = False`. This file is generated by the Remix developer menu when the user changes settings in-game. It overrides `rtx.conf` because Remix loads config layers in order: `dxvk.conf → rtx.conf → user.conf`, with later values winning.

This was actually documented in SUMMARY.md from build 016 but was never fixed in the game directory's `user.conf`. Every build from 016 to 074 was testing with replacement assets disabled.

The 8 building mesh hashes in mod.usda were captured at an unknown prior configuration (likely before SHORT4→FLOAT3 expansion, before texcoord fix, and/or before `positions` was added to the hash rule). They need to be re-captured with the current proxy and Remix settings.
