Title: "🧪 Add tests for compute_md5 in build_validator"

Description:
🎯 **What:** The testing gap addressed
The `compute_md5` function in `automation/build_validator.py` lacked test coverage. This stateless utility function is straightforward to test with mock files but critical for deterministic artifact hash validation.

📊 **Coverage:** What scenarios are now tested
1.  **Empty file:** Ensures hashing an empty byte array works as expected and compares correctly to the `b""` MD5 hash.
2.  **Short text content:** Tests basic string hashing against a known `bytes` hash value.
3.  **Large file content:** Creates a file large enough to trigger the chunked read `for` loop, ensuring chunk iteration works properly on large files.
4.  **Non-existent files:** Explicitly tests that `FileNotFoundError` is raised when the input file path does not exist.

✨ **Result:** The improvement in test coverage
The stateless artifact hashing behavior of `automation/build_validator.py:compute_md5` is fully covered against `pytest`, increasing test safety for continuous integration pipeline modifications and preventing silent failures in build validation rules.
